# Grafana-ascent
## Formation de 3 jours
### Introduction 
Le paysage technologique contemporain est marqué par une explosion exponentielle de la télémétrie, 
où la capacité d'une organisation à interpréter ses données en temps réel définit sa résilience opérationnelle. 
Au centre de cet écosystème se trouve Grafana, une plateforme devenue l'outil de référence pour la visualisation et
l'analyse des données de séries temporelles.1 Initialement conçu comme une simple couche de visualisation, 
Grafana a évolué pour devenir ce que les experts qualifient désormais de véritable « système d'exploitation de 
l'observabilité ».

### Genèse et Philosophie : L'Émergence du Modèle « Big Tent »
L'histoire de Grafana commence en 2014, lorsque Torkel Ödegaard décide de 
créer un fork de Kibana. À cette époque, Kibana s'oriente de plus 
en plus exclusivement vers l'écosystème Elastic Stack, délaissant 
le support pour d'autres sources de données essentielles comme InfluxDB 
et Graphite. La vision d'Ödegaard était radicalement différente : 
il s'agissait de construire une plateforme agnostique capable de requêter 
les données là où elles résident, sans imposer de migration vers une base 
de données propriétaire.
